Théorie des types homotopiques

De manière générale, une théorie ayant pour but d'être une fondation des mathématiques est construite sur deux couches :

  • La théorie logique dans laquelle nous pouvons écrire des propositions.


  • La théorie dans laquelle est exprimée notre axiomatique.
Cependant, la théorie des types est unique en son genre puisqu'elle constitue son propre système déductif. Un type correspond ainsi à la fois à une proposition et à une construction algébrique.

De manière générale, la théorie des types peut être considérée comme un système d'inférences qui peut être combiné pour déduire des propositions. \[ \frac{\mathcal{J}_1 \hspace{10pt} \mathcal{J}_2 \hspace{10pt} \dots \hspace{10pt} \mathcal{J}_n}{C} \] Les règles d'inférences contiennent :
  • au-dessus de la séparation une liste finie $\mathcal{J}_1,\ \mathcal{J}_2,\ \dots,\ \mathcal{J}_n$ de jugements qui correspondent aux prémices de notre argument.


  • en-dessous de la séparation, un seul jugement $C$ correspondant à notre conclusion.


Un exemple trivial de règle d'inférence est celle stipulant que dans n'importe quel context $\Gamma$ nous pouvons utiliser un élément $a:A$ et une fonction $f : A \rightarrow B$ pour obtenir un élément $f(a) : B$. \[ \frac{\Gamma \vdash a:A \hspace{10pt} \Gamma \vdash f : A \rightarrow B}{\Gamma \vdash f(a) : B} \]



La notion d'égalité est également différente puisqu'elle suppose l'existence d'une égalité propositionnelle $=$ et jugementale $\equiv$. Ainsi, une égalité propositionelle est un type (ou un $\infty$-groupoïde avec l'interpretation homotopique) et ses éléments sont considérés comme des preuves ou des "témoins" de l'identification d'un élément avec un autre.

En accord avec l'équivalence type/proposition, la proposition stipulant que pour n'importe quels élémets $a, b:A$ dans un context $\Gamma$ soit égaux est également un type. \[ \frac{\Gamma \vdash A : \mathcal{U}_i \hspace{10pt} \Gamma \vdash a:A \hspace{10pt} \Gamma \vdash b:B}{\Gamma \vdash a =_{A} b : \mathcal{U}_i} \] On introduit ainsi la fonction $\mathrm{refl}$ en tant qu'élément de base du type identité \[ \frac{\Gamma \vdash A : \mathcal{U}_i \hspace{10pt} \Gamma \vdash a:A}{\Gamma \vdash \mathrm{relf}_a : a=_{A} a} \]



En considérant que une section $b$ d'une famille $B$ indexée sur $A$ dans le contexte $\Gamma$, c'est à dire en considérant \[ \Gamma,\ x:A \vdash b(x) : B(x) \] D'une certaine manière, une telle section $b$ peut-être vue comme un operateur d'assignement $x \mapsto b(x)$ ou un programme prenant en entrée $x:A$ et retourne un terme $b(x): B(x)$ où le type $B(x)$ dépend de l'argument $x:A$.



Le type des toutes les fonctions dépendantes est appelé le type de fonction dépendantes \[ \frac{\Gamma,\ x:A \vdash B(x)\ type}{\Gamma \vdash \displaystyle \prod_{x:A} B(x)} \] Le principe de récursion d'un type $A$ est une fonction dépendante retournant une fonciton prenant en paramètre un élément $x:A$. Le principe de récursion pour le type $\mathrm{Id}_{A}$ permet ainsi de démontrer l'indiscernabilité des identités. \[ \frac{\Gamma,\ x:A \vdash C(x)\ \mathrm{type}}{\Gamma \vdash \displaystyle \prod_{x,y:A} \prod_{p:x=_{A}y} C(x) \rightarrow C(y)} \]

Cela nous indique que pour n'importe quelle famille de type $C$ repectant une égalité, dans le sens oû appliquer $C$ à des éléments égaux de $A$ résultent églement en une fonction entre les types résultants. De manière générale, la fonction $f(x,y,p) : C(x) \rightarrow C(y$ est toujours une equivalence de types et dans l'interpretation univalente une équivalence homotopique.



De la même manière, nous pouvons généraliser le principe de récursion pour les fonctions dépendantes : c'est le principe d'induction qui dans le cas du type $\mathrm{Id}_A$ peut être vu comme le fait d'asserter que la famille de $\mathrm{Id}_A$ peut être générée librement par les éléments de la forme $\mathrm{refl}_x : x=x$ \[ \mathrm{ind}_{=_{A}} : \prod_{C: \prod_{x,y:A} (x =_{A} y) \rightarrow \mathcal{U}} \left( \prod_{x:A} C(x, x, \mathrm{refl}_{x}) \right) \rightarrow \prod_{x,y:A} \prod_{p : x=_{A} y} C(x, y, p) \] defini par \[ \mathrm{ind}_{=_{A}}(C, c, x, x, \mathrm{refl}_x) :\equiv c(x) \] En particulier, le principe des chemins d'induction nous indique que la famille des types $(x =_A y)$, avec $x$ et $y$ variant sur les elements de $A$ est defini inductivement par les elements de la forme $\mathrm{refl}_{x}$. Ainsi, donner un element de n'importe quelle autre famille $C(x, y, p)$ dependante sur un element generique $(x, y, p)$ de la famille d'identite, il suffit de considerer les cas de la forme $(x, x, \mathrm{refl}_x)$.

Dans l'interpretation homotopique, on en deduit que le type des triples $(x, y, p)$ ou $x$ et $y$ sont les points d'arrivee d'un chemin $p$ (en outre, le type $\sum_{(x,y:A)} (x=_{A}y)$), est inductivement genere par une boucle constante pour chaque point $x$.

Interpretation homotopique

En theorie classique homotopique, a espace $X$ est un ensemble de points equipe avec une topologie, et ou un chemin entre chaque point $x$ et $y$ est representee par une application lineaire $p : [0,1] \rightarrow X$ avec $p(0) = x$ et $p(1) = y$. Parfois, une egalite stricte n'est pas forcement utile dans la mesure ou les operations naturelles definies sur des chemins ne s'appliquent plus. La solution est donc d'utiliser une relation d'equivalence homotopique.

Une homotopie entre une paire d'applications $f : X_1 \rightarrow X_2$ et $g : X_1 \rightarrow X_2$ est une application $h : X_1 \times [0,1] \rightarrow X_2$ satisfaisant $h(x, 0) = f(x)$ et $h(x, 1) = g(x)$.

Puisque une homotopie est une relation d'equivalence respectant des operations telles que la concatenation, ou l'inverse, les classes equivalences homotopiques de boucles en un point $x_0$ forme un groupe appele le groupe fondamentale, invariant algebriquement.
